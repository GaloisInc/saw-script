    4 {- |
    5 Module      : CryptolSAWCore.Pretty
    6 Copyright   : Galois, Inc. 2026

    9 Stability   : experimental
   10 
   11 Wrapper around Cryptol's prettyprinter.
   12 
   13 Cryptol and SAW have, regrettably, entirely incompatible
   14 prettyprinters. They both use the @Prettyprinter@ package, but there
   15 are at least two points of basic incompatibility: Cryptol wraps the
   16 package in a monad to avoid needing to carry the printing options
   17 around, which SAW doesn't; then even if it didn't do that, it also
   18 uses a different set of annotations so the type of `Doc ann` wouldn't
   19 match.
   20 
   21 There's also an impedance mismatch because SAW's convention is (now)
   22 to use `pretty` to produce docs and `pp` to produce strings, and
   23 Cryptol's convention is for some reason the other way around.
   24 
   25 This module wraps Cryptol's prettyprinter so that other code doesn't
   26 have to worry about these complications.
   27 
   28 It is possible that in the long run we should change SAW's
   29 prettyprinter so it also carries its print options around monadically.
   30 It is definitely a headache to pass that value around all over the
   31 place; in Cryptol's model you only need the options at render time and
   32 can interrogate them freely when actually printing.
   33 
   34 For the time being we just render all Cryptol `Doc`s to text before
   35 returning them to SAW's prettyprinter. In the FUTURE we should
   36 probably possible to open Cryptol's monadic doc abstraction and pass
   37 in the printing options to get a plain `Doc` back, and then by
   38 converting annotations (currently Cryptol's are not representable in
   39 SAW's, but that's easily fixed) we can return real `Doc`s out. This
   40 is definitely better for having the output come out well.
   41 
   42 This module is intended to be imported qualified, like this:
   43 ```
   44 import qualified CryptolSAWCore.Pretty as CryPP
   45 ```
   46 
   47 -}

   76 -- | Pretty-print a Cryptol value to a `Doc`.
   77 --   As per the notes above, converts to `Text` first.
   78 --
   79 --   For now, print to a generic doc because we can; when/if we start
   80 --   passing through Cryptol's docs and converting them to SAW docs
   81 --   we'll need to change this. Postpone it for the moment because it
   82 --   has a fairly large downstream footprint.
   83 pretty :: (Cry.PP a) => a -> PP.Doc ann
   84 pretty x = PP.pretty $ pp x
   85 
   86 -- | Pretty-print a Cryptol value to `Text`.
   87 pp :: (Cry.PP a) => a -> Text
   88 pp x =
   89     -- Note: Cry.pp produces a Cryptol `Doc`, not text.
   90     let doc = Cry.pp x in
   91     -- For now, just use the default render via Cryptol's Show instance.
   92     -- XXX: there are actually printing options and we should pass in
   93     -- SAW's user settings where they apply (and where they don't, maybe
   94     -- we should grow a few more options, like for printing floats).
   95     Text.pack $ show doc
   96 
   97 -- | Print using a `NameMap`
   98 prettyWithNames :: Cry.PP (Cry.WithNames a) => Cry.NameMap -> a -> PP.Doc ann
   99 prettyWithNames names x =
  100     PP.pretty $ Text.pack $ show $ Cry.ppWithNames names x

  102 -- | Call Cryptol's `addTNames`. This is a function that provides
  103 --   default names for type parameters that don't have any, and wouldn't
  104 --   need to be here except it interacts directly with the prettyprinter
  105 --   (hence the prettyprinter config) and is what you use to produce the
  106 --   `NameMap` for calling `ppWithNames`.
  107 --
  108 --   FUTURE: maybe replace `Cry.defaultPPCfg` with a config built
  109 --   from SAW's printing config.
  110 --
  111 addTNames :: [CryType.TParam] -> Cry.NameMap -> Cry.NameMap
  112 addTNames tparams names =
  113     CryType.addTNames Cry.defaultPPCfg tparams names
