    1 {- |
    2 Module      : SAWCentral.Bisimulation
    3 Description : Implementations of SAW-Script bisimulation prover
    4 License     : BSD3
    5 Maintainer  : bboston7
    6 Stability   : experimental
    7 
    8 This module provides tools to prove bisimilarity of two circuits, or of a
    9 circuit and a specification. At the moment, it does this through the single
   10 'proveBisimulation' function, but we will expand this module with additional
   11 functionality in the future.
   12 
   13 At its core, we want to prove that two circuits executing in lockstep satisfy
   14 some relation over the state of each circuit and their outputs. To achieve this,
   15 the 'proveBisimulation' command takes:
   16 * A list of already proved bisimulation theorems
   17 * A state relation @srel : lhsState -> rhsState -> Bit@
   18 * An output relation @orel : (lhsState, output) -> (rhsState, output) -> Bit@
   19 * A term @lhs : (lhsState, input) -> (lhsState, output)@
   20 * A term @rhs : (rhsState, input) -> (rhsState, output)@
   21 and considers @lhs@ and @rhs@ bisimilar when the following two theorems hold:
   22 * OUTPUT RELATION THEOREM:
   23     forall s1 s2 in.
   24       srel s1 s2 -> orel (lhs (s1, in)) (rhs (s2, in))
   25 * STATE RELATION THEOREM:
   26     forall s1 s2 out1 out2.
   27       orel (s1, out1) (s2, out2) -> srel s1 s2
   28 
   29 The OUTPUT RELATION THEOREM ensures that if the state relation holds prior to
   30 executing @lhs@ and @rhs@, then the output relation holds after executing @lhs@
   31 and @rhs@.  That is, the two terms step together.
   32 
   33 The STATE RELATION THEOREM ensures that if the output relation holds over some
   34 state and output value, then the state relation also holds over that same state.
   35 This ensures that the output relation captures the state relation.
   36 
   37 To enable compositionality, 'proveBisimulation' takes a list of proven
   38 bisimulation theorems as 'BisimTheorem's.  For each 'BisimTheorem', this
   39 implementation searches for invocations of the functions in the 'BisimTheorem'
   40 and replaces them with uninterpreted values that satisfy the 'BisimTheorem''s
   41 output relation.  Put another way, if @g_lhs@ contains some function @f_lhs@ and
   42 @g_rhs@ contains some function @f_rhs@ where @f_lhs@ and @f_rhs@ satisfy some
   43 output relation @f_orel@, then in the proof of some OUTPUT RELATION THEOREM for
   44 @g_lhs@ and @g_rhs@, the prover can replace @f_lhs@ with an uninterpreted value
   45 @v_lhs@ and @f_rhs@ with an uninterpreted value @v_rhs@.  Lastly, the prover
   46 assumes @v_lhs@ and @v_rhs@ satisfy @f_orel@.
   47 
   48 In order to make this assumption about @v_lhs@ and @v_rhs@, the state relation
   49 of the outer terms must ensure that the inner terms' state relation holds.  As
   50 such, the prover generates and proves a side condition for each applied
   51 'BisimTheorem'.  Let @g_lhs_s@ and @g_rhs_s@ be the states for @g_lhs@ and
   52 @g_rhs@ respectively.  Additionally, let there be a function
   53 @extract_inner_state x x_s y@ that takes an outer term @x@, an outer state
   54 @x_s@, and an inner term @y@, and returns the inner state of @x_s@ that @x@
   55 passes to @y@.  Lastly, let @g_srel@ be the state relation for the @g@ terms,
   56 and @f_srel@ be the state relation for the @f@ terms.  The prover then checks:
   57   COMPOSITION SIDE CONDITION:
   58     forall g_lhs_s g_rhs_s.
   59       g_srel g_lhs_s g_rhs_s -> f_srel f_lhs_s f_rhs_s
   60       where
   61         f_lhs_s = extract_inner_state g_lhs g_lhs_s f_lhs
   62         f_rhs_s = extract_inner_state g_rhs g_rhs_s f_rhs
   63 
   64 The reason for all of this complexity around composition is ultimately to reduce
   65 the burden on the SMT solver by uninterpreting functions so that the SMT solver
   66 may handle more complex proofs.
   67 -}

  129 -- | Translate a list of 'TypedTerm' representing theorems for input to the
  130 -- given validity-checking script and attempt to prove them.
  131 proveAll :: ProofScript () -> [TypedTerm] -> TopLevel ()
  132 proveAll script ts = do
  133   sc <- getSharedContext
  134   pos <- getPosition
  135   forM_ (zip [0..] ts) $ \(n, t) -> do
  136     io $ checkBooleanSchema (ttType t)
  137     prop <- io $ predicateToProp sc Universal (ttTerm t)
  138     let goal = ProofGoal
  139               { goalNum  = n
  140               , goalType = "prove"
  141               , goalName = "prove_bisim"
  142               , goalLoc  = show pos
  143               , goalDesc = ""
  144               , goalSequent = propToSequent prop
  145               , goalTags = mempty
  146               }
  147     res <- runProofScript script prop goal Nothing "prove_bisim" True False
  148     case res of
  149       UnfinishedProof {} -> failProof res
  150       ValidProof _ thm -> recordProof thm
  151       InvalidProof {} ->  failProof res
  152   where
  153     failProof :: ProofResult -> TopLevel ()
  154     failProof res = do
  155       opts <- rwPPOpts <$> getTopLevelRW
  156       fail $ "prove_bisim failed.\n" ++ showsProofResult opts res ""

  159 scRelation :: TypedTerm -> Term -> Term -> TopLevel Term
  160 scRelation rel relLhs relRhs = do

  162   io $ scApplyAll sc (ttTerm rel) [relLhs, relRhs]

  164 -- | Build the COMPOSITION SIDE CONDITION for 'bc' and 'bt'.  See the
  165 -- documentation at the top of this file for information on the COMPOSITION SIDE
  166 -- CONDITION.
  167 --
  168 -- This function works by examining the specific calls to @f_lhs@ in @g_lhs@ and
  169 -- @f_rhs@ in @g_rhs@ to deduce how these subfunctions access state in their
  170 -- respective super functions.  It then extracts these accessors and
  171 -- instantiates them with the specific @g_lhs_s@ and @g_rhs_s@ values passed
  172 -- into @g_lhs@ and @g_rhs@ respectively.
  173 buildCompositionSideCondition
  174   :: BisimComponents
  175   -- ^ Components of the outer bisimulation under verification
  176   -> BisimTheorem
  177   -- ^ Bisimulation theorem concerning inner function
  178   -> TopLevel TypedTerm
  179 buildCompositionSideCondition bc innerBt = do
  180   sc <- getSharedContext
  181   let outerBt = bcTheorem bc
  182 
  183   lhsOuterState <- io $ scLocalVar sc 0        -- g_lhs_s
  184   rhsOuterState <- io $ scLocalVar sc 1        -- g_rhs_s
  185 
  186   -- NOTE: Although not used in the final formula, we need to capture the input
  187   -- to the outer functions because the extracted inner function applications
  188   -- depend on it.  Therefore, it is necessary to match the expected form of the
  189   -- inner ExtCns that this function instantiates.
  190   input    <- io $ scLocalVar sc 2        -- in
  191 
  192   -- Locate inner function calls on each side and replace their arguments with
  193   -- 'ExtCns's
  194   (lhsInnerEc, lhsInnerApp) <-
  195     openConstantApp (bisimTheoremLhs innerBt) (bisimTheoremLhs outerBt)
  196   (rhsInnerEc, rhsInnerApp) <-
  197     openConstantApp (bisimTheoremRhs innerBt) (bisimTheoremRhs outerBt)
  198 
  199   -- Extract state accessors from each inner function
  200   lhsInnerState <- stateFromApp lhsInnerApp  -- f_lhs_s
  201   rhsInnerState <- stateFromApp rhsInnerApp  -- f_rhs_s
  202 
  203   -- Outer state relation
  204   -- g_srel g_lhs_s g_rhs_s
  205   outerRel <-
  206     scRelation (bisimTheoremStateRelation outerBt) lhsOuterState rhsOuterState
  207 
  208   -- Inner state relation
  209   -- f_srel f_lhs_s f_rhs_s
  210   innerRel <-
  211     scRelation (bisimTheoremStateRelation innerBt) lhsInnerState rhsInnerState
  212 
  213   -- Replace extcns in inner relation with outer inputs
  214   lhsTuple <- io $ scTuple sc [lhsOuterState, input]  -- (f_lhs_s, in)
  215   rhsTuple <- io $ scTuple sc [rhsOuterState, input]  -- (f_rhs_s, in)
  216   innerRel' <- io $
  217     scInstantiateExt sc
  218                      (Map.fromList [ (ecVarIndex lhsInnerEc, lhsTuple)
  219                                    , (ecVarIndex rhsInnerEc, rhsTuple)])
  220                      innerRel
  221 
  222   -- outer state relation implies inner state relation
  223   -- g_srel g_lhs_s g_rhs_s -> f_srel f_lhs_s f_rhs_s
  224   implication <- io $ scImplies sc outerRel innerRel'
  225 
  226   -- Theorem to prove. Note that the 'input' is ultimately unused (see NOTE on
  227   -- 'input' at the top of this function).
  228   -- forall g_lhs_s g_rhs_s. g_srel g_lhs_s g_rhs_s -> f_srel f_lhs_s f_rhs_s
  229   args <- io $ mapM
  230     (\(name, t) -> (name,) <$> C.importType sc C.emptyEnv t)
  231     [ ("input", bcInputType bc)
  232     , ("rhsState", bisimTheoremRhsStateType outerBt)
  233     , ("lhsState", bisimTheoremLhsStateType outerBt) ]
  234   theorem <- io $ scLambdaList sc args implication
  235   io $ mkTypedTerm sc theorem

  237 -- | Extract the state from the 'App' within a bisimulation side. Fails if 'app'
  238 -- is not an 'App'.
  239 stateFromApp :: TermF Term -> TopLevel Term
  240 stateFromApp app = do
  241   sc <- getSharedContext
  242   case app of
  243     App _ arg -> io $ scFlatTermF sc $ PairLeft arg
  244     _ -> do
  245       term <- io $ scTermF sc app
  246       fail $ "Error: " ++ showTerm term ++ " is not an App"

  248 -- | Given a term containing the application of a 'Constant', locate this
  249 -- application and replace its argument with an 'ExtCns'.  Returns the inserted
  250 -- 'ExtCns' and the updated 'App'.  Fails if 'constant' is not a 'Constant'.
  251 openConstantApp :: TypedTerm
  252                 -- ^ 'Constant' to search for.
  253                 -> TypedTerm
  254                 -- ^ 'TypeTerm' to locate 'Constant' in.  Must itself be a
  255                 -- 'Constant' (will be unfolded).
  256                 -> TopLevel (ExtCns Term, TermF Term)
  257 openConstantApp constant t = do
  258   sc <- getSharedContext
  259   -- Unfold constant
  260   name <- Text.unpack <$> constantName (unwrapTermF (ttTerm t))
  261   tUnfolded <- unfold_term [name] t
  262 
  263   -- Break down lambda into its component parts.
  264   (nm, tp, body) <- lambdaOrFail tUnfolded
  265 
  266   -- Replace outer function's argument with an 'ExtCns'
  267   -- NOTE: The bisimulation relation type ensures this is a single argument
  268   -- lambda, so it's OK to apply scOpenTerm once and not recurse
  269   (ec, tExtconsified) <- io $ scOpenTerm sc nm tp 0 body
  270   extractedF <- extractApp constant tExtconsified
  271   pure (ec, extractedF)
  272 
  273   where
  274     -- Break down lambda into its component parts.  Fails if 'tt' is not a
  275     -- lambda.
  276     lambdaOrFail :: TypedTerm -> TopLevel (LocalName, Term, Term)
  277     lambdaOrFail tt =
  278       case asLambda (ttTerm tt) of
  279         Just lambda -> return lambda
  280         Nothing ->
  281           fail $ "Error: Expected a lambda term, got " ++ show (ppTypedTerm tt)
  282 
  283 -- Traverses 'term' and extracts the application of 'constant' within it.  Fails
  284 -- if 'constant' cannot be found within 'term'.
  285 extractApp :: TypedTerm
  286            -- ^ 'Constant' to search for.
  287            -> Term
  288            -- ^ 'Term' to search in.
  289            -> TopLevel (TermF Term)
  290 extractApp constant term =
  291   case snd $ go (IntSet.empty, Nothing) term of
  292     Just app -> return app
  293     Nothing -> fail $ unlines [ "Error: Failed to locate constant in term."
  294                               , "  Constant: " ++ show (ppTypedTerm constant)
  295                               , "  Term: " ++ showTerm term ]
  296   where
  297     go :: (IntSet.IntSet, Maybe (TermF Term))
  298        -> Term
  299        -> (IntSet.IntSet, Maybe (TermF Term))
  300     go (seen, acc) t =
  301       case acc of
  302         Just res -> (seen, Just res)
  303         Nothing ->
  304           case t of
  305             Unshared tf -> termf (seen, acc) tf
  306             STApp{ stAppIndex = i, stAppTermF = tf } ->
  307               if IntSet.member i seen
  308               then (seen, acc)
  309               else termf (IntSet.insert i seen, acc) tf
  310 
  311     termf :: (IntSet.IntSet, Maybe (TermF Term))
  312           -> TermF Term
  313           -> (IntSet.IntSet, Maybe (TermF Term))
  314     termf (seen, acc) tf =
  315       case tf of
  316         App fn _ | unwrapTermF fn == unwrapTermF (ttTerm constant) ->
  317           (seen, Just tf)
  318         _ -> foldl' go (seen, acc) tf

  320 -- | Apply a prior proof result to simplify a bisimulation 'Term'.  Returns an
  321 -- updated 'Term', and a potential side condition that must be proven (or
  322 -- 'Nothing' if simplification had no effect).
  323 --
  324 -- This function works by searching for the functions from 'bt' in 'term'.  If
  325 -- it finds them, it replaces the calls in 'term' with 'ExtCns's and adds an
  326 -- assumption that the output relation holds over those 'ExtCns's.  This has the
  327 -- effect of saving the SMT solver from having to reason again about
  328 -- subrelations that have already been proven.  More formally, if @g_1@ and
  329 -- @g_2@ contain subfunctions @f_1@ and @f_2@ that satisfy some output relation
  330 -- @orel@, then this function replaces @f_1@ and @f_2@ with the uninterpreted
  331 -- values @v_1@ and @v_2@ respectively and adds the assumption @orel v_1 v_2@.
  332 --
  333 -- When simplification succeeds, this function also returns the side condition
  334 -- that the outer state relation from @g_1@ and @g_2@ implies that the inner
  335 -- state relation on @f_1@ and @f_2@ holds.  See the documentation on
  336 -- 'buildSuperStateRelation' for more information on the side condition.
  337 applyTheorem :: BisimTheorem
  338              -- ^ Theorem to apply
  339              -> BisimComponents
  340              -- ^ Components of bisimulation under verification
  341              -> Term
  342              -- ^ Term to simplify
  343              -> TopLevel (Term, Maybe TypedTerm)
  344 applyTheorem bt bc term0 = do
  345   sc <- getSharedContext
  346 
  347   -- Attempt to replace calls in 'term0' to the left side of 'bt' with an
  348   -- 'ExtCns'.  In other words, replace @f_1@ with @v_1@.
  349   let tpLhs = C.TCon (C.TC (C.TCTuple 2)) [ bisimTheoremLhsStateType bt
  350                                           , bisimTheoremOutputType bt ]
  351   (term1, lhsRs) <-
  352     State.runStateT (replaceConstantTerm (bisimTheoremLhs bt) tpLhs term0)
  353                     emptyReplaceState
  354 
  355   -- Attempt to replace calls in 'term1' to the right side of 'bt' with an
  356   -- 'ExtCns'.  In other words, replace @f_2@ with @v_2@.
  357   let tpRhs = C.TCon (C.TC (C.TCTuple 2)) [ bisimTheoremRhsStateType bt
  358                                           , bisimTheoremOutputType bt ]
  359   (term2, rhsRs) <-
  360     State.runStateT (replaceConstantTerm (bisimTheoremRhs bt) tpRhs term1)
  361                     emptyReplaceState
  362 
  363   case (rsExtCns lhsRs, rsExtCns rhsRs) of
  364     (Just lhsEc, Just rhsEc) -> do
  365       -- Simplification succeeded! Add assumption and generate side condition
  366 
  367       -- Apply inner output relation to 'ExtCns's
  368       -- orel v_1 v_2
  369       app <- io $ scApplyAll sc
  370                               (ttTerm (bisimTheoremOutputRelation bt))
  371                               [ lhsEc , rhsEc ]
  372 
  373       -- Use implication to assume relation holds in the main condition
  374       -- orel v_1 v_2 -> term2
  375       main <- io $ scImplies sc app term2
  376 
  377       -- Build side condition
  378       side <- buildCompositionSideCondition bc bt
  379 
  380       pure (main, Just side)
  381     _ ->
  382       -- Application failed to match on one or both sides.  Return the Term
  383       -- unchanged.
  384       pure (term0, Nothing)
  385 
  386 -- | Apply multiple proof results to simplify a bisimulation 'Term'.  Returns an
  387 -- updated 'Term' and a list of side conditions that must be proven.  See the
  388 -- documentation for 'applyTheorem' for more info.
  389 applyAllTheorems :: [BisimTheorem]
  390                  -- ^ Theorems to apply
  391                  -> BisimComponents
  392                  -- ^ Components of bisimulation under verification
  393                  -> Term
  394                  -- ^ Term to simplify
  395                  -> TopLevel (Term, [TypedTerm])
  396 applyAllTheorems bthms bc term = foldM go (term, []) bthms
  397   where
  398     go :: (Term, [TypedTerm]) -> BisimTheorem -> TopLevel (Term, [TypedTerm])
  399     go (curTerm, sides) bt = do
  400       (nextTerm, mSideCondition) <- applyTheorem bt bc curTerm
  401       case mSideCondition of
  402         Nothing ->
  403           -- Application failed
  404           pure (nextTerm, sides)
  405         Just sideCondition -> pure (nextTerm, sideCondition : sides)

  408 -- | Build the OUTPUT RELATION THEOREM for 'bc', as well as COMPOSITION SIDE
  409 -- CONDITION terms for any theorems used in the OUTPUT RELATION THEOREM.
  410 -- Returns a list of TypedTerms that must be proved. See the documentation at
  411 -- the top of this file for information on OUTPUT RELATION THEOREM and
  412 -- COMPOSITION SIDE CONDITION.
  413 buildOutputRelationTheorem :: [BisimTheorem]
  414                            -- ^ Theorems available to use in proof
  415                            -> BisimComponents
  416                            -- ^ Components of bisimulation under verification
  417                            -> TopLevel [TypedTerm]
  418 buildOutputRelationTheorem bthms bc = do
  419   sc <- getSharedContext
  420   let outerBt = bcTheorem bc
  421 
  422   -- Outer function inputs. See comments to the right of each line to see how
  423   -- they line up with the documentation at the top of this file.
  424   lhsState <- io $ scLocalVar sc 0        -- s1
  425   rhsState <- io $ scLocalVar sc 1        -- s2
  426   input <- io $ scLocalVar sc 2           -- in
  427 
  428   -- LHS/RHS inputs
  429   lhsTuple <- io $ scTuple sc [lhsState, input]  -- (s1, in)
  430   rhsTuple <- io $ scTuple sc [rhsState, input]  -- (s2, in)
  431 
  432   -- LHS/RHS outputs
  433   -- lhs (s1, in)
  434   lhsOutput <- io $ scApply sc (ttTerm (bisimTheoremLhs outerBt)) lhsTuple
  435   -- rhs (s2, in)
  436   rhsOutput <- io $ scApply sc (ttTerm (bisimTheoremRhs outerBt)) rhsTuple
  437 
  438   -- Initial relation result
  439   -- srel s1 s2
  440   initRelation <-
  441     scRelation (bisimTheoremStateRelation outerBt) lhsState rhsState
  442 
  443   -- Relation over outputs
  444   -- orel (lhs (s1, in)) (rhs (s2, in))
  445   relationRes <-
  446     scRelation (bisimTheoremOutputRelation outerBt) lhsOutput rhsOutput
  447 
  448   -- initRelation implies relationRes
  449   -- srel s1 s2 -> orel (lhs (s1, in)) (rhs (s2, in))
  450   implication <- io $ scImplies sc initRelation relationRes
  451 
  452   -- Simplify Term with any theorems that apply
  453   (implication', sideConditions) <- applyAllTheorems bthms bc implication
  454 
  455   -- Function to prove
  456   -- forall s1 s2 in out1 out2.
  457   --   srel s1 s2 -> orel (lhs (s1, in)) (rhs (s2, in))
  458   args <- io $ mapM
  459     (\(name, t) -> (name,) <$> C.importType sc C.emptyEnv t)
  460     [ ("input", bcInputType bc)
  461     , ("rhsState", bisimTheoremRhsStateType outerBt)
  462     , ("lhsState", bisimTheoremLhsStateType outerBt) ]
  463   theorem <- io $ scLambdaList sc args implication'
  464 
  465   tt <- io $ mkTypedTerm sc theorem
  466 
  467   pure $ tt : sideConditions
  468 
  469 -- | Build the STATE RELATION THEOREM for 'bc'. See the documentation at the top
  470 -- of this file for information on the STATE RELATION THEOREM.
  471 buildStateRelationTheorem :: BisimComponents -> TopLevel TypedTerm
  472 buildStateRelationTheorem bc = do
  473   sc <- getSharedContext
  474   let outerBt = bcTheorem bc
  475 
  476   -- Outer function inputs. See comments to the right of each line to see how
  477   -- they line up with the documentation at the top of this file.
  478   lhsState <- io $ scLocalVar sc 0        -- s1
  479   rhsState <- io $ scLocalVar sc 1        -- s2
  480   initLhsOutput <- io $ scLocalVar sc 2   -- out1
  481   initRhsOutput <- io $ scLocalVar sc 3   -- out2
  482 
  483   -- LHS/RHS initial outputs
  484   lhsTuple <- io $ scTuple sc [lhsState, initLhsOutput]  -- (s1, out1)
  485   rhsTuple <- io $ scTuple sc [rhsState, initRhsOutput]  -- (s2, out2)
  486 
  487   -- LHS of implication
  488   -- orel (s1, out1) (s2, out2)
  489   lhsRelation <-
  490     scRelation (bisimTheoremOutputRelation outerBt) lhsTuple rhsTuple
  491 
  492   -- RHS of implication
  493   -- srel s1 s2
  494   rhsRelation <-
  495     scRelation (bisimTheoremStateRelation outerBt) lhsState rhsState
  496 
  497   -- lhsRelation implies rhsRelation
  498   -- orel (s1, out1) (s2, out2) -> srel s1 s2
  499   implication <- io $ scImplies sc lhsRelation rhsRelation
  500 
  501   -- Function to prove
  502   -- forall s1 s2 in out1 out2.
  503   --   orel (s1, out1) (s2, out2) -> srel s1 s2
  504   args <- io $ mapM
  505     (\(name, t) -> (name,) <$> C.importType sc C.emptyEnv t)
  506     [ ("initRhsOutput", bisimTheoremOutputType outerBt)
  507     , ("initLhsOutput", bisimTheoremOutputType outerBt)
  508     , ("rhsState", bisimTheoremRhsStateType outerBt)
  509     , ("lhsState", bisimTheoremLhsStateType outerBt) ]
  510   theorem <- io $ scLambdaList sc args implication
  511 
  512   io $ mkTypedTerm sc theorem

  514 -- | Use bisimulation to prove that two terms simulate each other.  Proves the
  515 -- OUTPUT RELATION THEOREM and STATE RELATION THEOREM for the bisimilar terms.
  516 -- Additionally enables compositionality through the 'bthms' parameter.  This
  517 -- function also proves the COMPOSITION SIDE CONDITION for every successfully
  518 -- applied theorem in 'bthms'.  See the documentation at the top of this file
  519 -- for the definitions of bisimulation and these theorems.
  520 proveBisimulation ::
  521   ProofScript () ->
  522   -- ^ Proof script to use over generated bisimulation term
  523   [BisimTheorem] ->
  524   -- ^ Previously proved theorems that may be applied
  525   TypedTerm ->
  526   -- ^ Relation over states for terms to prove bisimilar. Must have type
  527   -- @lhsState -> rhsState -> Bit@  Also called the "state relation".
  528   TypedTerm ->
  529   -- ^ Relation over states and outputs for terms to prove bisimilar. Must have
  530   -- type @(lhsState, output) -> (rhsState, output) -> Bit@.  Also called the
  531   -- "output relation".
  532   TypedTerm ->
  533   -- ^ LHS of bisimulation. Must have type
  534   -- @(lhsState, input) -> (lhsState, output)@
  535   TypedTerm ->
  536   -- ^ RHS of bisimulation. Must have type
  537   -- @(rhsState, input) -> (rhsState, output)@
  538   TopLevel BisimTheorem
  539 proveBisimulation script bthms srel orel lhs rhs = do
  540   -- Typechecking
  541   (lhsStateType, rhsStateType, outputType) <- typecheckOutputRelation
  542   typecheckStateRelation lhsStateType rhsStateType
  543   (lhsName, lhsInputType) <- typecheckSide lhs lhsStateType outputType
  544   (rhsName, rhsInputType) <- typecheckSide rhs rhsStateType outputType
  545   unless (lhsInputType == rhsInputType) $
  546     fail $ unlines [ "Error: Mismatched input types in bisimulation terms."
  547                    , "  LHS input type: " ++ C.pretty lhsInputType
  548                    , "  RHS input type: " ++ C.pretty rhsInputType ]
  549 
  550   let bt = BisimTheorem
  551            { bisimTheoremStateRelation = srel
  552            , bisimTheoremOutputRelation = orel
  553            , bisimTheoremLhs = lhs
  554            , bisimTheoremRhs = rhs
  555            , bisimTheoremOutputType = outputType
  556            , bisimTheoremLhsStateType = lhsStateType
  557            , bisimTheoremRhsStateType = rhsStateType
  558            }
  559 
  560   let bc = BisimComponents
  561            { bcTheorem = bt
  562            , bcInputType = lhsInputType
  563            }
  564   outputTheorems <- buildOutputRelationTheorem bthms bc
  565   stateTheorem <- buildStateRelationTheorem bc
  566   proveAll script $ stateTheorem : outputTheorems
  567 
  568   -- Proof succeeded!
  569   printOutLnTop Info $ concat [ "Successfully proved bisimulation between "
  570                               , Text.unpack lhsName
  571                               , " and "
  572                               , Text.unpack rhsName ]
  573 
  574   return $ BisimTheorem srel orel lhs rhs outputType lhsStateType rhsStateType
  575 
  576   where
  577     -- Typecheck output relation. The expected type is:
  578     -- @(lhsStateType, outputType) -> (rhsStateType, outputType) -> Bit@
  579     --
  580     -- If the relation typechecks, 'typecheckOutputRelation' evaluates to a
  581     -- tuple of:
  582     -- @(lhsStateType, rhsStateType, outputType)@
  583     -- Otherwise, this invokes 'fail' with a description of the specific
  584     -- typechecking error.
  585     typecheckOutputRelation :: TopLevel (C.Type, C.Type, C.Type)
  586     typecheckOutputRelation =
  587       case ttType orel of
  588         TypedTermSchema
  589           (C.Forall
  590             []
  591             []
  592             (C.TCon
  593               (C.TC C.TCFun)
  594               [ C.TCon (C.TC (C.TCTuple 2)) [s1, o1]
  595               , C.TCon
  596                 (C.TC C.TCFun)
  597                 [ C.TCon (C.TC (C.TCTuple 2)) [s2, o2]
  598                 , C.TCon (C.TC C.TCBit) []]])) -> do
  599           unless (o1 == o2) $ fail $ unlines
  600             [ "Error: Mismatched output types in relation."
  601             , "LHS output type: " ++ C.pretty o1
  602             , "RHS output type: " ++ C.pretty o2 ]
  603 
  604           return (s1, s2, o1)
  605         _ -> fail $ "Error: Unexpected output relation type: "
  606                  ++ show (ppTypedTermType (ttType orel))
  607 
  608     -- Check that 'lhsStateType' and 'rhsStateType' match the extracted types
  609     -- from 'typecheckOutputRelation'.  Invokes 'fail' if the types do not
  610     -- match.
  611     typecheckStateRelation :: C.Type -> C.Type -> TopLevel ()
  612     typecheckStateRelation lhsStateType rhsStateType =
  613       case ttType srel of
  614         TypedTermSchema
  615           (C.Forall
  616             []
  617             []
  618             (C.TCon
  619               (C.TC C.TCFun)
  620               [ s1
  621               , C.TCon
  622                 (C.TC C.TCFun)
  623                 [ s2, C.TCon (C.TC C.TCBit) []]])) -> do
  624           unless (s1 == lhsStateType) $ fail $ unlines
  625             [ "Error: LHS of state relation and output relations have incompatible state types:"
  626             , "  State relation LHS state type: " ++ C.pretty s1
  627             , "  Output relation LHS state type: " ++ C.pretty lhsStateType ]
  628 
  629           unless (s2 == rhsStateType) $ fail $ unlines
  630             [ "Error: RHS of state relation and output relations have incompatible state types:"
  631             , "  State relation RHS state type: " ++ C.pretty s2
  632             , "  Output relation RHS state type: " ++ C.pretty rhsStateType ]
  633         _ -> fail $ "Error: Unexpected state relation type: "
  634                  ++ show (ppTypedTermType (ttType srel))
  635 
  636     -- Typecheck bisimulation term. The expected type for a bisimulation term
  637     -- is:
  638     -- @(stateType, inputType) -> (stateType, outputType)@
  639     --
  640     -- If the term typechecks, this function returns a pair containing the name
  641     -- of the 'Constant' and @inputType@.  Otherwise, this function invokes
  642     -- 'fail' with a description of the specific typechecking error.
  643     typecheckSide
  644       :: TypedTerm -> C.Type -> C.Type -> TopLevel (Text.Text, C.Type)
  645     typecheckSide side stateType outputType = do
  646       -- Check that 'side' is a 'Constant' (necessary for composition)
  647       name <- constantName $ unwrapTermF $ ttTerm side
  648 
  649       -- Check function arguments
  650       case ttType side of
  651         TypedTermSchema
  652           (C.Forall
  653             []
  654             []
  655             (C.TCon
  656               (C.TC C.TCFun)
  657               [ C.TCon (C.TC (C.TCTuple 2)) [s, i]
  658               , C.TCon (C.TC (C.TCTuple 2)) [s', o] ])) -> do
  659           unless (s == stateType) $ fail $ unlines
  660             [ "Error: State type in bisimulation term input does not match state type in relation."
  661             , "  Expected: " ++ C.pretty stateType
  662             , "  Actual: " ++ C.pretty s]
  663 
  664           unless (s' == stateType) $ fail $ unlines
  665             [ "Error: State type in bisimulation term output does not match state type in relation."
  666             , "  Expected: " ++ C.pretty stateType
  667             , "  Actual: " ++ C.pretty s']
  668 
  669           unless (o == outputType) $ fail $ unlines
  670             [ "Error: Output type in bisimulation term does not match output type in relation."
  671             ,"  Expected: " ++ C.pretty outputType
  672             , "  Actual: " ++ C.pretty o ]
  673 
  674           return (name, i)
  675         _ ->
  676           let stStr = C.pretty stateType in
  677           fail $ unlines
  678           [ "Error: Unexpected bisimulation term type."
  679           , "  Expected: (" ++ stStr ++ ", inputType) -> (" ++ stStr ++ ", outputType)"
  680           , "  Actual: " ++ show (ppTypedTermType (ttType side)) ]
  681 
  682 -- | Replace the invocation of a specific 'Constant' with an 'ExtCns'.  The
  683 -- function returns the resulting 'Term' and updates a 'ReplaceState' to hold
  684 -- the generated 'ExtCns' and the specific 'App' that was replaced.
  685 replaceConstantTerm :: TypedTerm
  686                     -- ^ 'Constant' to replace application of
  687                     -> C.Type
  688                     -- ^ 'constant's return type
  689                     -> Term
  690                     -- ^ 'Term' to perform replacement in
  691                     -> State.StateT ReplaceState TopLevel Term
  692 replaceConstantTerm constant constantRetType term = do
  693   sc <- lift getSharedContext
  694   case term of
  695     Unshared termF -> do
  696       termF' <- replaceConstantTermF termF
  697       liftIO $ scTermF sc termF'
  698     STApp{ stAppIndex = i, stAppTermF = termF } -> do
  699       table <- State.gets rsMemo
  700       case Map.lookup i table of
  701         Just x -> return x
  702         Nothing -> do
  703           termF' <- replaceConstantTermF termF
  704           term' <- liftIO $ scTermF sc termF'
  705           let table' = Map.insert i term' table
  706           State.modify$ \st -> st { rsMemo = table' }
  707           return term'
  708   where
  709     -- | Partner function to 'replaceConstantTerm' that operates over 'TermF's.
  710     replaceConstantTermF
  711       :: TermF Term -> State.StateT ReplaceState TopLevel (TermF Term)
  712     replaceConstantTermF termF = do
  713       case termF of
  714         App x _ | unwrapTermF x == unwrapTermF (ttTerm constant) ->
  715           State.gets rsExtCns >>= \case
  716             Just v ->
  717               State.gets rsApp >>= \case
  718                 Just a | a == termF ->
  719                   -- Encountered another call to the function under replacement
  720                   -- that matches the replaced function.  This can happen even
  721                   -- when the underlying Cryptol does not explicitly make
  722                   -- multiple calls because translation to SAWCore can insert
  723                   -- additional function calls with the same arguments.  In this
  724                   -- case, simply return the same 'ExtCns' already generated.
  725                   return $ unwrapTermF v
  726                 Just _ -> do
  727                   -- Encountered a call to the function under replacement with
  728                   -- different arguments.  This isn't yet supported.
  729                   name <- Text.unpack <$> lift (constantName (unwrapTermF x))
  730                   fail $ concat ["Error: Encountered multiple calls to "
  731                                 , name
  732                                 , ". Use of composition with multiple "
  733                                 , "subfunction calls is not yet supported."
  734                                 ]
  735                 _ -> panic "replaceConstantTermF"
  736                            ["rsApp should always exist when rsExtCns exists"]
  737             Nothing -> do
  738               sc <- lift getSharedContext
  739 
  740               -- Generate an 'ExtCns' and return it, thereby replacing 'termF'
  741               -- with it.
  742               tp <- liftIO $ C.importType sc C.emptyEnv constantRetType
  743               name <- lift $ constantName $ unwrapTermF x
  744               v <- liftIO $ scFreshGlobal sc name tp
  745               State.modify $ \st -> st { rsExtCns = Just v, rsApp = Just termF }
  746               return $ unwrapTermF v
  747         _ ->
  748           -- Recurse
  749           mapM (replaceConstantTerm constant constantRetType) termF
  750 
  751 -- Extract the name from a 'Constant'. Fails if provided another kind of 'TermF'
  752 constantName :: TermF Term -> TopLevel Text.Text
  753 constantName (Constant e _) = return $ toShortName $ ecName e
  754 constantName tf = do
  755   sc <- getSharedContext
  756   term <- io $ scTermF sc tf
  757   fail $ "Error: Expected a constant, but got: " ++ showTerm term
